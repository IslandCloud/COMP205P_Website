Some terminology
  1. Random Sample: The random sample with a scalar is a method where we randomly select 1000 samples and send this to the front end along with a scalar multiple so the number can be multiplied back up to the real set.
  2. JSON Splitting: This is where we split the string with delimiters and count the resulting array. It happens to be the size of the set.
  3. Dictionary Size: This is the value of parsing the JSON string into a Dictionary Object and returning the size of the Dictionary.

Unit Tests

We have created a few unit tests for the Set Engine. This is done by adding an optional parameter to the GenerateSets() function in the Set Engine which puts the function into test mode. The Set Engine then reads from a Test Database and the output is tested against a set of known output sets by calling the CompareSets(TestSet, PredictedSet) function where 'TestSet' is the answer and 'PredictedSet' is the set generated by the Set Engine. Every single item in the Test and Predicted Set must match in order for the unit test to pass.
