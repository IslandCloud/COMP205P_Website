User Actions

These are API calls that an end user will find useful when using this API. These are actions that provide data from the DB in various formats, and are the gateway to accessing and filtering this data.

Getting started

Using this API, you will be able to receive sets of data, generally in the form of a Key/Value pair. The key will be the GUID of the element, and the Value will be the property (if relevant) associated with that element and type of request.

For example, if you request all "Students" and the property that interests you is the "Gender", then you will receive Key/Value pairs like this:

Guid: "eb24a5df-515a-42c9-878c-986e2ed214bb"
Value: "M"

Requesting Dummy Data

The way to request dummy data is to use the /api/SetRequest/?set={set name} suffix. For example, let's say there is a set that is called 'lorem.txt' (such a file exists on the server for testing purposes, so feel free to use this.)


You could request this set by using the suffix: /api/SetRequest/?set=lorem.txt

Getting zipped results

When requesting a set using the /api/SetRequest/?set={set name} suffix, an optional zip parameter can be added to tell the API to return the data in a zipped file. This significantly reduces the size of the data and hence reduces the tranfer time. This zip will need to be unzipped client-side. An example of this would be:

/api/SetRequest/?set=lorem.txt&zip=true

Backend Helper Actions (Used by Team 2 during development)

Using the suffix /api/Team2Actions/{Action}/{parameters}, Team 2 is able to execute some useful actions to aid in the testing and development of the project.

GenerateStudents/{number:int}

Using the suffix /api/Team2Actions/GenerateStudents/{number:int}, where {number:int} is an integer > 0, we can generate random entries in the "Students" table in the Dummy DB, which include the parameters: Guid, Name, Gender, MathsGrade, CompGrade.
